# 人生调音台

一个 5 分钟自我显影网页小游戏。

它首先是一个适合转发的社交小游戏，其次才是个人成长工具。玩家通过心动采样、底噪消除和未来试音，生成一份《我的个人使用说明 v0.1》。

## 当前 MVP

- 电子人生调音台 + 漫画小剧场视觉风格
- BPM 小狗吉祥物和三视图角色设定
- 18 张心动采样题，配 6 张统一风格漫画场景
- 12 组底噪二选一
- 8 个未来身份
- 自动生成个人使用说明
- 结果分享卡、下载专辑分享卡、复制朋友圈文案
- 两首可切换 CC0 完整背景音乐 + Web Audio 交互音效

## 最近一版迭代

- 接入 OpenGameArt 的两首 CC0 音乐：`Two Simple Game Music Loops` 的 menu loop 和 `Flowerbed Fields`
- 顶部 HUD 新增音乐选择按钮，可在 Simple Loop / Flowerbed 两种 BGM 间切换，并记住玩家上次选择
- 背景配乐改为真实 OGG 循环音频，Web Audio 保留为按钮、选择、结果揭晓等交互音效层
- 结果页新增“朋友盲猜卡”：自动生成 A/B/C 三个可晒外号，让朋友先猜哪个才是真正结果
- 下载分享卡和复制朋友圈文案同步加入盲猜选项，把单向晒结果升级成轻互动挑战
- 增加 `vercel.json` 静态部署配置，方便发布到 Vercel 预览环境
- 顶部 HUD 新增音乐音量滑杆，支持 0-120% 实时调节，并记住玩家上次设置
- 将默认音乐音量略微抬高，同时保留轻音乐的留白感，避免开局听感过轻
- 升级题库为“漫画短剧版”：心动采样从能力描述改成更有画面的生活瞬间，提高“这不就是我吗”的命中感
- 底噪二选一扩展为 24 组现代社交/工作刺点，加入群聊、会议、临时改需求、隐形付出等更容易会心一笑的场景
- 未来身份新增“准确标签 + 可晒外号”，例如“人间翻译器 / 清晰解释者”“混乱降噪师 / 可靠组织者”
- 结果页和分享卡新增“人生 BGM”描述，把音乐也变成可转发的人格线索
- 音乐升级为“幸福轻音乐版”：78 BPM、五声音阶、轻快但留白，整体更像被 BPM 小狗陪着慢慢调音
- 新增空气感回声层和八音盒式高频点缀，让页面有一点小小的闪光感，但不持续轰炸玩家
- 主循环从连续拨片改为 kalimba / soft music-box 质感的短句，低频脉冲只在关键阶段轻轻托底
- 重写交互音效：靠近是上扬小笑声，无感是柔和确认，远离是温柔下行，不再给人压迫或判定感
- 结果揭晓增加 4 秒左右的温暖收尾，让生成《个人使用说明》的瞬间更有分享仪式感
- 升级为《人生调音台 0.3：采样器版》：心动采样页更像一台可玩的声音设备
- 新增实时声纹画像，玩家每次选择都会改变右侧声纹轮廓
- BPM 小狗增加状态演出：靠近、旁路、静音时会切换不同监听状态
- 结果页从“说明书报告”升级为“人生专辑”，生成更适合晒出的专辑名
- 重写三张分享卡：猜主旋律、一句话使用说明、朋友如何打开我
- 基于舒适型游戏音乐和 UI 音效调研，重做音乐混音方向：更慢、更稀疏、更有留白
- 将背景循环从高密度拨片改为 72 BPM 的呼吸型短旋律，减少持续压迫感
- 降低低频脉冲和交互和弦密度，心动选择只保留轻柔的 1-2 个音符反馈
- 修复音乐按钮显示“音乐开”但尚未真正解锁播放的问题，未播放前改为“开声音”
- 点击“开始调音”会明确开启音乐，并提高整体音量和启动提示音的可感知度
- 心动采样页压缩卡片高度，答题按钮贴近底部，减少每题都要下滑的打断感
- 同一阶段内换题不再强制滚回页面顶部，连续答题更顺
- 新增灵动、有趣的程序化背景音乐，不依赖外部音频文件
- 顶部 HUD 增加音乐开关，玩家可以随时打开或静音
- 心动采样、底噪消除、未来试音、结果揭晓都有不同音效反馈
- 不同游戏阶段会自动调整音乐能量，保持“电子调音台”的沉浸感
- 修复底噪二选一和未来身份选择的快速点击重复触发问题
- 修复没有选择“靠近”时结果页点亮列表为空的问题
- 修复 MBTI 参考文案在提示为空时出现空句的问题
- 修复复制朋友圈文案时会覆盖页面小狗评语的问题
- 增加分享卡下载的图片加载容错和按钮状态反馈
- 将题目数量、选择数量、动效延迟和进度段集中到 `GAME_CONFIG`
- 随机抽样改为 Fisher-Yates 洗牌，避免随机排序偏斜
- 新增 BPM 小狗校准耳机的短过场，让开局更像进入游戏
- 心动采样拆成三段节奏提示，减少连续答题疲劳
- 选择后按钮会短暂锁定，保留“亮轨”反馈的手感
- 结果页新增一句高光结论，先给玩家可截图、可转发的判断
- 继续保留“不是诊断、不是人格判决”的产品边界

## 工程升级建议

- 下一步可以把 `index.html` 拆成 `data.js`、`engine.js`、`share.js` 和 `styles.css`
- 题库继续扩展时，建议给每道题增加 `id`、`dimension`、`comic`、`tone` 字段，方便做 AB 测试
- 分享玩法继续升级时，建议先做“朋友圈猜我卡”和“双人同频报告”，不要急着加复杂登录系统
- 图片资源现在比较大，正式传播前建议压缩成 WebP/AVIF，并保留 PNG 兜底
- 如果未来换成真实音乐资产，可以保留当前 Web Audio 音效作为按钮反馈和阶段转场

## UI/UX 升级方向

- 顶部改成游戏 HUD：品牌、当前阶段、进度条同时可见
- 首屏增加三张体验承诺卡，明确这是小游戏、社交测试和说明书
- 背景增加克制网格和更细的光感，提升电子录音棚质感
- 操作按钮按选择语义分色，并加入 hover/pressed/disabled 状态
- 底噪二选一增加“更不能忍”的决策提示，强化对抗感
- 结果分享卡加强专辑封面感，让结果更像能被发出去的作品

## 产品原则

- 不做心理诊断
- 不输出固定人格标签
- 不把 MBTI 当结论
- 用具体生活场景和边界选择提高“准”的感觉
- 让玩家愿意继续点、愿意截图、愿意发给朋友
